# Beyond OS v41-31.2 — 멘토링 좌석배치도 표시 보정

기준선: v41-31.1

## 반영 내용

1. 좌석배치도 멘토링 표시 안정화
   - 오늘 요일의 활성 멘토링 차시와 활성 배정을 대시보드 API에서 다시 조회합니다.
   - 차시 시작 10분 전부터 차시 종료 전까지 해당 학생 좌석에 파란색 `멘토링` 배지가 표시됩니다.
   - Supabase nested select 관계명 문제로 `mentoring_slots`가 비어도 표시가 누락되지 않도록, 오늘의 차시를 먼저 조회한 뒤 배정값에 차시 정보를 수동으로 붙입니다.

2. 좌석 매칭 보강
   - 입실 세션의 student_id, 좌석의 current_student_id, 학생 기본좌석 default_seat_no를 모두 후보로 사용합니다.
   - 학생이 아직 미입실 상태여도 기본 좌석이 지정되어 있으면 좌석배치도에서 멘토링 예정 표시가 잡히도록 보강했습니다.
   - 관리주의 빨간색이 필요한 경우에는 기존처럼 빨간색을 우선하고, `멘토링` 배지는 보조로 표시됩니다.

3. 멘토링 배정 변경 후 즉시 갱신
   - 멘토링 차시 저장, 기본 세팅, 학생 배정, 배정 삭제, 차시 비활성화 후 대시보드 데이터를 즉시 새로 불러옵니다.
   - 새로 배정한 학생이 조건 시간대에 들어오면 좌석배치도에 바로 반영되도록 했습니다.
