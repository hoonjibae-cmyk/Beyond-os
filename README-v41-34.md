# Beyond OS v41-34 — 멘토링 운영 UI 정리 및 충돌 방지 개선

기준선: v41-33.4

## 반영 사항

1. 개인일정 저장 시 멘토링 충돌 경고
   - 학생 시간표/액티비티 시간표 저장 전, 해당 날짜에 이미 잡힌 멘토링 일정과 수정하려는 개인 일정이 맞는지 검증합니다.
   - 멘토링 시간이 체류 예정 시간 밖이거나 외출 시간과 겹치면 경고 확인창을 표시합니다.
   - 운영 예외가 있을 수 있으므로, 경고 확인 후 그래도 저장은 가능합니다.

2. 학생 배정 패널 담당학생 우선 정렬
   - 멘토를 선택하면 선택된 학생, 담당학생이 목록 위로 먼저 올라오도록 정렬을 보강했습니다.
   - 담당학생은 기존처럼 흰색, 비담당학생은 회색으로 표시됩니다.

3. 날짜별 일정에서 학생 카드 드래그 이동
   - 날짜별 일정이 `일정 수정 시작` 상태일 때만 학생 카드에 `이동` 핸들이 표시됩니다.
   - 데스크탑에서 `이동` 핸들을 잡고 다른 차시 카드로 드래그하면 해당 날짜에 한해 차시를 옮길 수 있습니다.
   - 드롭 후에도 개인일정 충돌 검사를 실행하며, 충돌 시 경고 확인 후 그래도 이동할 수 있습니다.

4. 멘토링 기본 설정을 설정 탭으로 분리
   - 멘토 이름/목표 담당학생 수/멘토별 담당학생 설정/1~8차시 기본 세팅/운영 기준 안내를 `설정 > 멘토링 기본 설정`으로 이동했습니다.
   - 멘토링 시간표 화면 상단은 날짜별 일정과 요일별 템플릿 운영에 집중하도록 간소화했습니다.

## 참고

- 드래그 이동은 날짜별 화면에서 `일정 수정 시작` 후에만 동작합니다.
- 요일 기본값 미리보기 상태에서는 드래그 이동, 배정 제외, 차시 제외가 모두 잠겨 있습니다.
- 실제 `next build`는 작업 환경의 `node_modules/next` 부재로 실행하지 못했고, API 라우트 JS 문법 검사와 `page.jsx` JSX 파싱 검사는 통과했습니다.
