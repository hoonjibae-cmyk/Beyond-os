# Beyond OS v41-15 — 데일리 리포트 마감 발송 콘솔

기준선: `beyond-os-v41-14-dashboard-cleanup-mobile-usability.zip`

## 핵심 변경

1. **데일리 리포트 마감 발송 콘솔 추가**
   - 위치: `데일리 리포트` 화면 상단
   - 정해진 마감시간에 전체 학생 리포트를 발송하기 전, 발송 가능/확인 필요/발송 불가 상태를 한 번에 점검합니다.

2. **전체 발송 기능 안정화**
   - 기존 `발송 가능 전체 발송`, `확인 필요 포함 발송` 버튼을 마감 발송 콘솔 안으로 정리했습니다.
   - 전체 발송 전 테스트모드/실전발송/Allowlist/Fail-safe 상태를 확인하도록 안내합니다.
   - 실제 보호자 전체 실전 발송 모드에서는 기존 안전확인 절차를 유지합니다.

3. **발송 전 점검 항목 요약**
   - 보호자 연락처 없음
   - 입실 기록 없음
   - 플래너 미제출
   - 오늘 코멘트 미입력
   - 순찰 없음
   - 입실시간 누락
   - 결석/지각/조퇴/외출과다/순공부족 등
   - 항목을 누르면 해당 학생 목록으로 바로 필터링됩니다.

4. **전체 발송 진행률 표시**
   - 전체 발송 실행 중 처리 인원 / 전체 인원 표시
   - 발송완료 / 발송대기 / 발송실패 집계 표시
   - 최근 처리 학생 결과 표시

5. **발송 결과 요약 및 실패 건 재발송**
   - 발송 종료 후 실패 건 수를 확인할 수 있습니다.
   - 실패한 학생만 다시 발송할 수 있는 버튼을 추가했습니다.

6. **리포트 발송 이력과 연동 유지**
   - 기존 `발송 이력` 패널은 유지했습니다.
   - 실패 로그 기준 재발송 기능도 그대로 유지됩니다.

## Supabase / 환경변수 / MacroDroid

추가 설정 없음.

- 신규 SQL 없음
- 신규 Vercel 환경변수 없음
- MacroDroid 변경 없음

기존 리포트 발송 관련 설정과 테스트모드 설정을 그대로 사용합니다.

## 검증 메모

`npm run build`에서 컴파일, TypeScript, 정적 페이지 생성 단계까지 정상 출력되었습니다. 다만 작업환경에서는 Next.js build 프로세스가 종료 신호를 반환하기 전에 타임아웃되어 명령 종료코드까지는 확인하지 못했습니다.
