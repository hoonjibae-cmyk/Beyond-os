# Beyond OS v41-37

## 배포 버전 배지 및 버전 단일 관리 (UI/UX 개선 기준 버전)

기준선: v41-36.2

### 반영 내용

1. 앱 헤더에 현재 배포 버전을 보여주는 `버전 v41-37` 배지를 추가했습니다.
   - 재배포 후 브라우저에서 `Ctrl+F5` 로 새로고침하면 배지 숫자로 새 버전이 실제로 반영됐는지 바로 확인할 수 있습니다.
   - 배지에 마우스를 올리면 버전 이름과 설명이 표시됩니다.

2. 화면에 표시되는 버전을 `lib/appVersion.js` 한 파일에서만 관리하도록 정리했습니다.

### 다음부터 버전 올리는 방법 (재배포 시)

1. `lib/appVersion.js` 파일을 엽니다.
2. `APP_VERSION` 값만 새 번호로 바꿉니다. (예: `'v41-37'` → `'v41-38'`)
   - 필요하면 `APP_VERSION_NAME`, `APP_VERSION_DESCRIPTION` 도 이번 배포 내용에 맞게 수정합니다.
3. (선택) `package.json` 의 `name`, `version` 도 같은 번호로 맞춰두면 관리가 편합니다. 단, 화면 표시에는 영향이 없습니다.
4. 아래 배포 명령을 실행합니다.

버전 표기 규칙 예시
- 큰 작업 라인: `v41-37`, `v41-38` ...
- 후속 패치: `v41-37.1`, `v41-37.2` ...

### 필수 SQL

없습니다.

### 배포

```powershell
npx.cmd vercel --prod --yes
```

배포 후 브라우저에서 `Ctrl+F5` 로 새로고침하면 헤더에 `버전 v41-37` 배지가 표시됩니다.

신규 환경변수 및 MacroDroid 변경은 없습니다.
