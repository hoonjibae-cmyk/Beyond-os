# Beyond OS v41-33.4 — 멘토링 날짜별 일정 안정화

## 기준 버전
- 기준: v41-33.3
- 목적: 날짜별 멘토링 일정 기능을 실제 운영 중 헷갈리지 않게 다듬고, 날짜별 제외/되돌리기/개인일정 주의 확인 흐름을 안정화했습니다.

## 반영 내용

### 1. 날짜별 일정 상태 배지 정리
날짜별 일정 차시 카드에 운영 상태가 더 명확히 표시됩니다.

- 요일 기본값
- 요일 기본값 기반
- 날짜별 수정됨
- 이 날짜에서만 추가됨
- 배정 변경 있음
- 개인일정 주의 있음

### 2. 개인일정 주의 상세 보기
학생 카드에 개인일정 주의가 표시되는 경우, 주의 표시 또는 `주의 상세` 버튼을 눌러 충돌 사유를 확인할 수 있습니다.

상세 팝업에는 다음 정보가 표시됩니다.

- 기준일
- 멘토링 시간
- 학생 개인 일정
- 시간표 메모
- 충돌 사유

### 3. 날짜별 제외 문구 정리
날짜별 일정에서의 삭제/비활성화는 요일 템플릿 삭제가 아니라 해당 날짜에서만 제외하는 의미이므로 문구를 정리했습니다.

- 학생 배정 삭제 → `이 날짜에서 제외`
- 차시 비활성화 → `이 날짜 차시 제외`

요일별 템플릿 화면에서는 기존 의미에 맞게 삭제/비활성화 문구를 유지합니다.

### 4. 중요 작업 확인창 보강
다음 작업은 실행 전 한 번 더 확인합니다.

- 일정 수정 시작
- 기본값 되돌리기
- 이 날짜에서 학생 제외
- 이 날짜 차시 제외

특히 기본값 되돌리기는 이 날짜에서만 추가·수정·제외한 차시와 배정 변경이 초기화된다는 점을 확인 문구에 명시했습니다.

### 5. 작업 피드백 강화
저장, 제외, 되돌리기 작업 중/완료/실패 메시지를 더 명확하게 표시합니다.

- 처리 중 메시지
- 완료 메시지
- 실패 시 새로고침 후 재시도 안내

# Beyond OS v41-33 — 멘토링 날짜별 일정 수정 및 개인 일정 충돌 표시

## 기준선

- 기준 버전: v41-32.3
- 패치 성격: 요일별 멘토링 템플릿을 날짜별 운영 일정으로 연결하고, 선택 날짜만 차시/배정을 수정할 수 있도록 보강

## 반영 내용

1. 멘토링 시간표 기본 화면을 `날짜별 일정` 중심으로 정리했습니다.
   - 오늘 날짜를 기본으로 열고, 해당 날짜의 요일에 맞는 요일별 멘토링 템플릿을 자동으로 보여줍니다.
   - 예: 오늘이 수요일이면 수요일 템플릿의 차시와 배정을 날짜별 화면에서 바로 확인합니다.
   - 상단에서 이전날/오늘/다음날, 날짜 직접 선택, 1주 달력 스트립으로 날짜를 이동할 수 있습니다.

2. 기존 `요일별 템플릿` 설정은 그대로 유지했습니다.
   - 월/수/금 기본 차시와 화/목 임시 차시 설정은 요일별 템플릿 화면에서 관리합니다.
   - 반복 배정도 요일별 템플릿 화면에서 처리합니다.

3. 선택 날짜만 별도로 수정할 수 있게 했습니다.
   - `선택 날짜 일정 수정 시작` 버튼을 누르면 해당 날짜의 요일 템플릿을 날짜별 일정으로 복사해 직접 수정할 수 있습니다.
   - 이후 차시 수정, 차시 비활성화, 학생 배정/삭제는 해당 날짜에만 적용됩니다.
   - `요일 기본값으로 되돌리기`를 누르면 해당 날짜의 직접 수정 내용을 삭제하고 현재 요일 템플릿 기준으로 다시 반영합니다.

4. 날짜별 일정이 좌석배치도 멘토링 표시와 연동됩니다.
   - 오늘 날짜에 별도 수정 일정이 있으면 좌석배치도 파란색 `멘토링` 표시는 날짜별 일정을 우선 사용합니다.
   - 오늘 별도 수정 일정이 없으면 기존 요일별 템플릿 기준으로 표시합니다.

5. 개인 일정 충돌 주의 표시를 학생 카드에 추가했습니다.
   - 멘토링 배정 후 학생 개인 시간표가 나중에 바뀐 경우에도, 멘토링 학생 카드에 `⚠ 개인일정 주의` 표시가 뜹니다.
   - 반대로 개인 일정이 확정된 상태에서 멘토링 차시나 배정을 변경해도 같은 기준으로 충돌을 표시합니다.
   - 충돌 기준은 개인 시간표의 학습실 체류 시간과 등록된 외출 시간입니다.

## 필수 SQL

이번 버전은 날짜별 멘토링 차시/배정을 저장하기 위한 신규 SQL이 필요합니다.

## 기준선

- 기준 버전: v41-32.2
- 패치 성격: 멘토링 시간표 → 출결·관리 이력 이동 시 조회기간 보정, GPT 상담 요약 순공 기준 보강

## 반영 내용

1. 멘토링 시간표에서 학생 카드 클릭 시 기본 조회기간 1주 적용

3. GPT 프롬프트 보강
   - 학생 피드백용/학부모 상담용 모두 위 순공 기준을 상담 요약에 반영하도록 안내했습니다.

   - 멘토링 대상임을 완전히 숨기지는 않고, 작은 보조 배지는 회색 톤으로 표시해 출결 상태와 혼동되지 않도록 했습니다.

2. 멘토링 시간표 UI/UX 정리
   - 멘토링 차시 카드 크기와 간격을 줄여 1~8차시 전체가 더 조밀하게 보이도록 조정했습니다.
   - 멘토 카드에는 `담당 학생 수`와 `시간표 배정 건수`를 분리해서 표시합니다.
   - 차시 카드, 배정 행, 우측 배정 패널의 여백과 폰트 크기를 정리했습니다.

3. 멘토별 담당학생 사전 설정 추가
   - 멘토링 시간표 탭에 `멘토별 담당학생 설정` 영역을 추가했습니다.
   - 관리자가 멘토별 담당학생을 미리 선택하고 저장할 수 있습니다.
   - 한 학생이 여러 멘토의 담당학생으로 동시에 보이지 않도록, 저장 시 해당 학생의 기존 다른 멘토 담당 연결은 자동 비활성화됩니다.

4. 학생 배정 시 담당/비담당 시각 구분
   - 배정 패널에서 멘토를 선택하면 해당 멘토의 담당학생은 흰색 카드로 표시됩니다.
   - 비담당학생은 회색 카드로 표시됩니다.
