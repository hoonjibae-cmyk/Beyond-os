# Beyond OS v41-31.4 — 멘토링 담당학생/좌석상태 우선순위 핫픽스

## 반영 내용

1. 좌석배치도 멘토링 표시 우선순위 보정
   - 멘토링 차시 시작 10분 전~차시 종료 구간이어도 학생이 `결석` 또는 `퇴실/조퇴` 상태라면 기존 출결 색상이 우선 표시됩니다.
   - 멘토링 대상임을 완전히 숨기지는 않고, 작은 보조 배지는 회색 톤으로 표시해 출결 상태와 혼동되지 않도록 했습니다.

2. 멘토링 시간표 UI/UX 정리
   - 멘토링 차시 카드 크기와 간격을 줄여 1~8차시 전체가 더 조밀하게 보이도록 조정했습니다.
   - 멘토 카드에는 `담당 학생 수`와 `시간표 배정 건수`를 분리해서 표시합니다.
   - 차시 카드, 배정 행, 우측 배정 패널의 여백과 폰트 크기를 정리했습니다.

3. 멘토별 담당학생 사전 설정 추가
   - 멘토링 시간표 탭에 `멘토별 담당학생 설정` 영역을 추가했습니다.
   - 관리자가 멘토별 담당학생을 미리 선택하고 저장할 수 있습니다.
   - 한 학생이 여러 멘토의 담당학생으로 동시에 보이지 않도록, 저장 시 해당 학생의 기존 다른 멘토 담당 연결은 자동 비활성화됩니다.

4. 학생 배정 시 담당/비담당 시각 구분
   - 배정 패널에서 멘토를 선택하면 해당 멘토의 담당학생은 흰색 카드로 표시됩니다.
   - 비담당학생은 회색 카드로 표시됩니다.
   - 비담당학생도 선택 및 배정은 가능합니다. 색상은 구분용입니다.

## 필수 SQL

Supabase SQL Editor에서 아래 파일을 실행해야 담당학생 설정 저장이 가능합니다.

```text

```

기존 v41-31 및 v41-31.3 SQL이 이미 적용되어 있어야 합니다.
